Yaël's Portfolio - Groups for Race for Life Project

Client: Cancer Research UK

Project Brief

Add the ability for race participants to fundraise and sign up for Race for Life in groups.

My Role

Freelance UX Consultant

Tools Used
  • Sketching
  • Wireframe documentation
The Challenge

Cancer Research run an annual event called Race for Life that is made up of many local races, whose participants run to race money to beat cancer.

Before 2012, to enter Race for Life, you could only enter as a sole participant or register another participant. There was no ability to be linked to other people and fundraise together as a group.

Cancer Research wanted to be able to integrate group registration within their current online registration form.

What I Did

I was briefed that the new Groups functionality would have to be included without changing the existing registration process for individuals, given me limited scope as to what I could and could not touch. This was partly due to the fact that the current registration process had been through extensive usability testing and multivariate testing already.

Evaluating the current registration process, as well as the technical limitations as discussed with the developers on the team, I began working on the user flows for group registration. This encompassed both creating a group and joining a group, and was achieved by mocking up the various flows using Omnigraffle.

Example user flow from the documentation I created

I then moved on to fleshing out the various screens - moving from sketches to wireframes in Omnigraffle. There were a number of considerations:

  • How to redesign the landing page so that both individual and group registration were represented
  • Furthermore, how to ensure that both people who wanted to create and join a group were represented
  • What group-specific copy would there need to be? What creator-specific and joiner-specific copy would there need to be?
  • How to adjust the current registration form to include group-specific questions where appropriate
  • How to improve the Thanks page once registered
  • How to accommodate people who abandon registration process
The Outcome

I created full documentation for this project, including user flows and wireframes for all the various scenarios and interactive states, in conjunction with close consultation from the Project Manager, Business Intelligence and Developer teams. I presented the wireframes to the stakeholders of the project, and with full approval, handed over the project ready for the Build phase.

Example wireframe from the documentation I created

Race For Life's Groups functionality was in place for Race for Life 2012.
